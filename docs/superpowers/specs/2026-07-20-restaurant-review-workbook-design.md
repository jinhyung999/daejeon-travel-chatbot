# 식당 후보 검토 워크북 설계

## 목적

`data/curation/restaurant_candidates.csv`의 460개 후보를 Excel에서 빠르게 검토하고 상태와 제외 사유를 기록할 수 있는 단일 `.xlsx` 워크북을 만든다. 원본 CSV는 변경하지 않으며 워크북을 사람 검토용 사본으로 사용한다.

## 출력

- 파일: `outputs/restaurant-review/restaurant_candidates_review.xlsx`
- 형식: 필터, 표, 수식, 데이터 유효성 검사와 조건부 서식이 포함된 Excel 워크북
- 시트: `검토 현황`, `후보 검토`, `검토 가이드`

## 검토 현황

- 전체 후보 수와 `pending`, `approved`, `hold`, `rejected` 상태별 개수를 수식으로 표시한다.
- 대덕구, 동구, 서구, 유성구, 중구별 전체 및 상태별 개수를 수식으로 표시한다.
- 상태 요약은 `후보 검토` 시트의 편집값을 참조해 자동 갱신한다.
- 제목, 안내문, 요약 카드, 구별 현황 표를 한 화면에서 읽을 수 있게 구성한다.

## 후보 검토

- 원본 21개 필드를 모두 보존한다.
- 검토 빈도가 높은 열을 앞쪽에 배치한다: 상태, 구, 상호명, 업종, 추천 점수, 추천 이유, 주소, 블로그 검색 링크, 중복 표시, 제외 사유.
- 좌표와 검색 횟수 같은 기술 열은 뒤쪽에 배치하되 삭제하지 않는다.
- 첫 행을 고정하고 전체 데이터 범위에 표 필터를 적용한다.
- `review_status`는 `pending`, `approved`, `hold`, `rejected`만 선택할 수 있는 드롭다운으로 만든다.
- 상태별 조건부 서식을 적용한다: 승인 초록, 보류 노랑, 제외 빨강, 미검토 회색.
- URL은 원문을 보존하고 클릭 가능한 링크로 표시한다.
- 긴 텍스트는 제한된 폭에서 줄바꿈하며, 행 높이와 열 너비를 검토에 적합하게 제한한다.

## 검토 가이드

- 상태값의 의미와 사용법을 설명한다.
- 대표 제외 사유를 제공한다: 기존 목록 중복, 폐업·휴업, 배달 위주, 일반 프랜차이즈, 비식당, 여행 추천 가치 낮음, 정보 부족.
- 추천 점수는 승인 판정이 아니라 검토 순서라는 점을 명시한다.
- 영업시간, 휴무일, 메뉴·가격, 주차, 예약 등은 수작업 확인 대상임을 안내한다.

## 데이터 처리

- CSV는 UTF-8로 읽는다.
- 점수와 검색 건수는 숫자로, 최신 게시일은 날짜로 변환한다.
- 빈 URL이나 빈 도로명 주소는 정상 공란으로 유지한다.
- CSV의 행 수와 각 구별 후보 수가 워크북과 일치해야 한다.

## 검증

- `후보 검토` 표가 헤더 포함 461행, 데이터 460행인지 확인한다.
- 구별 합계가 대덕구 100, 동구 100, 서구 100, 유성구 100, 중구 60인지 확인한다.
- 상태 수식 합계가 전체 후보 수와 일치하는지 확인한다.
- 수식 오류를 검색한다.
- 세 시트를 각각 렌더링하여 제목, 요약 수치, 헤더, 링크와 안내문이 잘리지 않는지 시각적으로 확인한다.
